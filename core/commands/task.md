Возьми в работу задачи: **$ARGUMENTS**

Сначала прочитай `.claude/forge.md` (команды доски) и `.claude/harness.json`
(каталог worktree). Затем для КАЖДОГО номера:

1. Прочитай задачу целиком командой из `forge.md`. Если на ней метка
   `needs-decision` или `blocked` — в работу НЕ бери: сначала реши вопрос
   с владельцем / дождись блокера.
2. Выбери роль-исполнителя: если задача чисто документационная и в
   `.claude/agents/` проекта есть свой документационный агент — он;
   всё, что меняет код, — агент `impl`. Модель бери из раздела «Модель» в теле
   задачи; если его нет — opus для `size:M`/`size:L`, sonnet для `size:S`.
3. Спауни агента через Agent tool с `isolation: "worktree"` — он получит
   изолированный git worktree и не столкнётся с другими агентами. В промпте
   передай: номер задачи, её полный текст, имя ветки и как связать PR с
   задачей (по `forge.md`). Ветка: есть код иерархии в титуле —
   `<type>/<код в нижнем регистре>-<слаг>` (`feat/m1-e2-t3-нормализация`),
   нет — `<type>/<слаг>-<N>`. Подпись (description) агента:
   `<код задачи или #N> · <модель> · <суть в 3–7 словах>` —
   например `M1-E2-T3 · opus · нормализация точек жестов`.
4. Несколько номеров — спауни всех агентов **одним сообщением**, параллельно.

Пока агенты работают, не правь те же файлы сам. Агент вернулся с нулём вызовов
инструментов — не перезапускай, а резюмни его через SendMessage: служебные
напоминания — фон, выполни бриф из первого сообщения инструментами.

Когда все вернутся — сводка: задача → PR → статус проверок → что требует
решения владельца. Ничего не мержи без команды `/land`.

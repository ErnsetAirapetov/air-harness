---
description: Сохранить состояние сессии перед сжатием контекста или паузой по лимиту
argument-hint: [limit — если остановка из-за 5-часового лимита]
---

Останови текущую работу и сохрани состояние. Причина: **$ARGUMENTS**
(пусто = контекст сессии подходит к пределу).

1. Перезапиши `.claude/handoff.md` (файл сессионный, в git не попадает):
   - дата и время; над чем шла работа и зачем (цель, номер задачи/PR);
   - фактическое состояние: ветки, PR, живые worktree, незакоммиченное
     (`git status`), запущенные и незавершённые агенты;
   - решения, принятые в этой сессии, и их причины;
   - следующий конкретный шаг — с чего начинать;
   - открытые вопросы владельцу.
   Пиши для агента, который этой сессии не видел: никаких «как обсуждали».
2. Если причина — `limit`: узнай время сброса лимита (его показывает Claude
   Code в сообщении о лимите; не видно — спроси владельца) и предложи команду
   планировщика ОС (schtasks на Windows, cron на Unix), запускающую в момент
   сброса из корня проекта:
   `claude -p "Прочитай .claude/handoff.md и продолжи работу с указанного шага"`.
   Команду покажи владельцу — сам планировщик не трогай без его «ок».
3. Иначе — скажи владельцу запустить `/compact` (или `/clear`, если история
   не нужна). После рестарта SessionStart-хук харнеса сам напомнит про хендофф.
4. После успешного продолжения в новой сессии хендофф удаляется — за это
   отвечает продолживший, не ты.

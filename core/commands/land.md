---
description: Отревьюить PR и, если чисто, замержить с полной уборкой
argument-hint: <номер PR> [ещё номера]
---

Проведи PR через ревью и мерж: **$ARGUMENTS**

Живые worktree: !`git worktree list`

Прочитай `.claude/forge.md` — команды PR и доски бери оттуда. Для каждого номера:

1. Спауни агента `reviewer` с номером PR. Несколько PR — всех одним
   сообщением, параллельно.
2. По вердикту ревьюера:
   - **МЁРЖИТЬ** → мерж командой из `forge.md`.
   - **ДОРАБОТАТЬ** → спауни `impl` в worktree этой ветки с промптом, который
     вернул ревьюер. Потом ревью заново.
   - **ПЕРЕДЕЛАТЬ** → не мержи; покажи владельцу разбор и спроси, что делать.
3. После успешного мержа обязательно убери за собой:
   - worktree ветки: `git worktree list --porcelain` → каталог с
     `branch refs/heads/<ветка>` → `git worktree remove <каталог>`
     (залоченное — `--force`, но только после мержа PR);
   - закрой задачу командой из `forge.md` с итогом и ссылкой на PR;
   - удали локальную ветку (`git branch -d <ветка>`).
4. `git worktree list` — покажи владельцу, что осиротевших деревьев не осталось.

В конце: сводка «замержено / на доработке / отклонено» и что осталось в бэклоге.
